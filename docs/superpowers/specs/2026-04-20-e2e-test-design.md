# AI-Gatekeeper E2E Test Design

**Date:** 2026-04-20

## Goal

Verify the full KBS↔plugin gRPC path end-to-end: JWT verification, OPA policy evaluation, mock Keycloak token exchange, and correct HTTP responses — using the same docker-compose approach as kbs-plugin-sdk-python.

## Structure

```
e2e/
├── Makefile
├── docker-compose.yml
├── Dockerfile.kbs               (builds KBS + kbs-client from Trustee source)
├── mock-keycloak.py             (single-file Python HTTP mock for Keycloak)
├── config/
│   ├── kbs.toml                 (KBS config with external plugin backend)
│   └── plugin-config.yaml       (plugin config with cert-based JWT verification)
├── policy.rego                  (test Rego policy)
└── tests/
    └── e2e.sh                   (test script)
```

## Services

| Service | Image | Purpose |
|---|---|---|
| `setup` | alpine/openssl | Generates RSA key pair (test-token.key + test-token-cert.pem) |
| `mock-keycloak` | python:3.12-slim | Returns `{"access_token":"mock-<scope>"}` on POST to token endpoint |
| `ai-gatekeeper` | built from repo | Plugin under test |
| `kbs` | built from Trustee | KBS with EXTERNAL_PLUGIN=true, external backend → ai-gatekeeper:50051 |

## Request Flow

```
test script
  → kbs-client (docker exec) → KBS        [get attestation token for KBS auth layer]
  → POST {"token": "<custom-jwt>"}
    Authorization: Bearer <kbs-token>
    → KBS → gRPC Handle() → plugin
      → verify custom-jwt with test-token-cert.pem
      → OPA eval → allow/deny
      → mock-keycloak POST → {"access_token": "mock-llama-8b"}
      → PluginResponse 200 {"endpoint": "...", "access_token": "..."}
    ← KBS returns response
  ← test script asserts
```

The plugin uses `token_cert_path` (not `jwks_url`) to avoid a circular dependency back to KBS. The custom JWT (role claim, sub claim) is signed by the test RSA key generated by `setup`.

## KBS Config

Insecure mode (`insecure_key = true`, `InsecureAllowAll`) for test. External plugin backend:
```toml
[[plugins]]
name = "external"
backends = [
  { name = "ai-gatekeeper", endpoint = "http://ai-gatekeeper:50051", tls_mode = "insecure", timeout_ms = 10000 },
]
```

## Test Cases

| Case | JWT role | Model | Expected |
|---|---|---|---|
| Happy path | basic | llama-8b | 200, body has endpoint + access_token |
| Policy deny | basic | llama-70b | 403 |
| Unknown model | premium | unknown-model | 404 |
| Missing token | — | llama-8b | 400 |

## Makefile Targets

- `make build` — build KBS + plugin images
- `make up` — start services detached
- `make test` — build → up → e2e.sh → down (exits non-zero on failure)
- `make down` — stop and remove containers
- `make logs` — follow container logs
